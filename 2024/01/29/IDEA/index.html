IDEA新建Maven项目

发表于 2024-01-29 更新于 2024-02-01 分类于 笔记

ctrl+alt+shift+s

如果创建的Spring Initialzr项目没有.iml文件

选择项目名 按两下ctrl 然后输入运行 mvn idea:module命令

解决Maven报错

Could not find artifact com.mysql:mysql-connector-j:pom:unknown in aliyunmaven

直接给mysql-connector-j指定版本

运行时报错

1.Invalid value type for attribute ‘factoryBeanObjectType’: java.lang.String

是由于mybatis依赖版本错误

2.Host ‘host.docker.internal‘ is not allowed to connect to this MySQL server

是由于搭建了docker环境 把项目放到docker环境内部了 和主机mysql隔离了 所以当docker项目访问mysql时 访问性质是局域网访问

解决方案：用navicat

然后刷新mysql权限

3.Could not find artifact org.springframework.cloud:spring-cloud-starter-netflix-eureka-client:pom

version不对

4.报错Error creating bean with name ‘scopedTarget.eurekaClient‘ defined in class

是由于spring-cloud-starter-netflix-eureka-client和spring-boot-starter-web两个依赖的时候 会出现冲突 由于代码里面 我用了Spring MVC的Rest方式 而没有用spring-cloud-starter-netflix-eureka-client本身包含Jesery Rest方式 导致出现下面的BUG

解决办法：

在eureka依赖里面排除Jersey，用SpringMVC Rest方式

5.Servlet.service() for servlet [dispatcherServlet] in context with path [] threw exception [Request processing failed: org.springframework.web.client.ResourceAccessException: I/O error on GET request for “http://userservice/user/1“: userservice] with root cause

java.net.UnknownHostException: userservice

这是由于忘记删service/impl里面的

6.如果报错Connection refused: connect

说明所有的微服务都挂掉了
